# TinyTalk

쉘 환경에서 명령어를 치면 TinyTalk이 내 환경에서 바로 실행 가능한 명령어로 바꿔줍니다.
사용자 환경을 스캔해서 실행 가능한 명령을 캐싱하고, 프롬트에 따라 알맞은 CLI Command를 출력합니다.
TinyTalk은 스스로 명령을 실행하지 않습니다. 실행 여부는 사용자가 결정할 수 있습니다.

![TinyTalk 데모](demo.gif)

```
? 디스크 제일 많이 먹는 것부터 보여줘

du -h -d1 / 2>/dev/null | sort -hr | head -20
↳ 상위 디렉터리 디스크 사용량, 큰 순서대로
```

마음에 들면 Enter, 아니면 고쳐서 실행하면 됩니다. 그게 전부예요.

## 로컬 에이전트로 빠르게 시작하기

클라우드 API 키가 없어도 됩니다. [Ollama](https://ollama.com)로 내 컴퓨터에서 도는 모델만으로도
충분히 잘 동작합니다.

```
brew install ollama
ollama pull qwen3:8b
./install.sh
```

설치 스크립트가 키가 필요 없는 로컬 서버를 자동으로 감지해서 `local` 백엔드로 바로 연결해줍니다.
API 키를 물어보지도 않고, 아무것도 외부로 나가지 않습니다. 그 다음부터는:

1. 빈 줄에서 `?`를 눌러 프롬프트 모드로 들어갑니다 (작은 `TinyTalk` 배지가 켜집니다).
2. 원하는 걸 한국어든 영어든 그냥 자연어로 쓰고 Enter.
3. TinyTalk이 내 셸에서 수집한 정보(실제로 깔려 있는 바이너리, OS, 셸 종류)를 기준으로 요청을
   검증하고, 입력창을 실제 명령어 + 한 줄 설명으로 바꿔치기합니다.
4. 읽어보고, 필요하면 고치고, 직접 실행합니다.

클라우드 백엔드(Claude, GPT 등)로 언제든 다시 바꿀 수 있습니다 — `~/.config/tinytalk/config.toml`의
`defaults.backend`만 바꾸면 됩니다. 로컬과 클라우드 백엔드를 같은 설정 파일에 같이 둘 수 있어서,
하나가 안 될 때 다른 쪽으로 넘어가게 구성해도 됩니다.

## 최신 벤치마크

TinyTalk은 자체 평가 스위트(`tt eval`)를 갖고 있습니다 — 자연어 명령 25개를 영어/한국어 양쪽으로
돌려서, 출력이 실제로 파싱되는지, 진짜 존재하는 바이너리를 쓰는지, 각 항목의 검증 조건을 통과하는지
채점합니다. 로컬 모델도 클라우드 모델과 나란히 포함되어 있어서, 완전히 오프라인으로 갔을 때 정확히
뭘 포기하게 되는지 한눈에 볼 수 있습니다.
