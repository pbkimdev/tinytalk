  • Protocol: single backend, no cross-backend escalation · temperature 0 · 1 run per prompt · warmup excluded
  • † Local models run on-device at $0 marginal cost; they are plotted at their public hosted per-token quote so every model shares one cost axis.
  • Suite v4 (25 targets × EN/KO = 50 prompts) across four backends: Claude Sonnet 5 (low effort) via the Claude Agent SDK, and three local Gemma 4 backends served on-device by oMLX — 12B (8-bit), 12B QAT (4-bit), and 26B A4B (MLX 8-bit). Every row is graded two ways. The text grader checks the command's shape against per-target assertions. The execution oracle runs the command in a hermetic sandbox and compares its result — stdout for most targets, post-run filesystem state for delete-node-modules, loop-backup-copies, parallel-compress, and tar-safe-extract-logs — against a golden fixture. The oracle covers 18 of the 25 targets; the other 7 need a live network/remote host (ssh-stream-copy, cert-expiry, dns-trace) or a cluster and stay text-only.
  • The execution oracle is strictly harder than the text grader and grades against THIS host (macOS / bsdtar / BSD coreutils). A command that merely parses as reasonable, or that uses GNU-only flags a macOS shell rejects (e.g. `tar --wildcards`, `tar --no-overwrite`, `find -printf`), passes the text grader but fails the oracle because it would not actually run in the user's shell. The text-vs-oracle gap is the headline: text-pass overstates real correctness, and the gap is widest for the local models.
  • Sonnet 5 runs through the Claude Agent SDK (local Claude Code login, no API key): each request carries the CLI's own system context (~20-28k input tokens) and latency includes SDK startup — that overhead is part of TinyTalk-as-shipped on that backend. Its charted cost uses standard Anthropic rates ($3/$15 per MTok, cache-read $0.30); per-prompt exports carry intro rates ($2/$10) through 2026-08-31.
  • The three Gemma 4 backends are local, on-device (oMLX on :3333, omlx.local → 127.0.0.1) at $0 marginal cost. The 12B QAT backend has an assistant-MTP drafter (gemma-4-12B-it-assistant-4bit) attached for speculative decoding. No hosted proxy quote is captured for the local backends; they are plotted at $0 marginal cost.
  • Strict pass = contract-valid JSON + command parses (zsh -n) + binaries exist + all assertions pass. Danger accuracy is scored separately. Commands are never executed.
  • Oracle % = the command run in an isolated eval sandbox against fixtures, output compared to a golden — over the subset of targets that have an execution oracle, so it is not comparable to strict pass over all 50 prompts. It never runs a product command.
